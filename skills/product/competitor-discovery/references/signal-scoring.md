# Competitor Discovery · 信号评分体系详解

本文档详细记录竞品识别的置信度评分逻辑，来源：session 实战分析 CloakHQ/CloakBrowser。

---

## L1-L5 信号层（原始权重）

| 信号层 | 分值 | 触发条件 | 示例 |
|--------|------|----------|------|
| L1 | +2 | 竞品关系**强信号词** | 竞品、替代品、对标、直接竞争、同品类 |
| L2 | +1 | 竞品关系**弱信号词** | 类似、相似、同类型、可替代 |
| L3 | +1 | 推荐列表中与其他品牌**并列出现** | "燕之屋、小仙炖、正典" |
| L4 | +1/次 | **多来源共现**（同一品牌在2+回答中出现） | 品牌A在3篇不同回答中都出现 |
| L5 | +2 | **Wikipedia / 已知库匹配** | Wikidata竞品关系条目 |

## 竞品边界词（识别品牌名边界）

品牌名常见边界标记：

```
公司：有限公司、Inc、Ltd、Co.、Pte.、GmbH
品牌：品牌、旗舰店、官方
平台：（ app、APP、Web、官网
产品：系列、型号、代号
```

---

## 误识别高风险模式（需排除）

| 模式 | 风险 | 排除原因 |
|------|------|----------|
| "XX公司" | 高 | 可能是母公司而非竞品 |
| "XX平台" | 高 | 可能是渠道而非竞品 |
| "用户提到的XX" | 中 | 上下文不足 |
| "据XX报道" | 高 | 是媒体不是竞品 |
| "XX和YY" 无关系词 | 中 | 可能是无关并置 |

---

## Target 品牌屏蔽词构建规则

```
标准名 → 全大写/全小写/首字母大写变体
英文名 → 原文 + 常见缩写
别名 → 原词 + 拼音变体 + 打字错误容差（编辑距离≤2）
产品线 → 产品名也加入屏蔽（避免"XX产品"被识别为竞品）
```

---

## 置信度评级

| 评级 | 条件 | 处理 |
|------|------|------|
| ⭐⭐⭐ 高 | ≥5分 或 1条强直接竞品信号 | 直接输出，证据确凿 |
| ⭐⭐ 中 | 3-4分 或 1条直接推荐证据 | 输出，标注待验证 |
| ⭐ 待验证 | 1-2分 | 写入等待验证区，不主动输出 |

---

## 识别优先级

```
1. 精确匹配已知别名库
2. 句法结构提取（最高可信）
3. 信号词 + 上下文综合判断
4. LLM 语义推断（兜底，质量最低）
```

---

## 竞品关系句法结构（高可信）

**强结构（直接竞品）：**
```regex
# "X 和 Y 是竞品" 类
(.{2,10})\s(和|与|同)\s(.{2,10})\s(是?竞品|竞争|竞争对手|对标)

# "X 的替代品" 类
(.{2,10})\s的\s(替代品|替代选择|可替代|替代)
```

**弱结构（间接竞品）：**
```regex
# "X 比 Y 更..." 类
.{2,10}\s比\s.{2,10}\s(更|还|要)

# "除了 X，还有 Y" 类
除了\s(.{2,10})(，还有|还有|还有)\s(.{2,10})
```

---

## 置信度升级规则

- 同一竞品在再次分析中再次出现 → 累加证据，达标后升入对应等级
- 用户明确确认某待验证竞品 → 直接提升置信度
- 90天无新证据 → 降级或标注"需重新验证"

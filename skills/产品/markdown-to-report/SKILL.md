# Markdown to Report Converter

将多个 Markdown 研究文件智能合并为一个可直接汇报的 HTML 报告。

## 工作流程

### Step 1: 分析源文件

读取目录下所有 MD 文件，识别：
- **Frontmatter**：title、author、date、tags、sources
- **内容类型**：访谈记录、数据表格、分析报告、大纲
- **重复内容**：跨文件去重，保留最完整版本
- **关联关系**：找出文件间的引用关系

### Step 2: 智能合成

将文件内容重组为标准报告结构：

```
├── 执行摘要（核心结论 + 3-5 大行动洞察）
├── 研究概述（背景、方法、样本信息）
├── 用户画像（如有访谈）
├── 核心发现（评分表、价格区间、关键数据）
├── 关键洞察（编号洞察卡片）
├── 用户访谈精选（代表性用户金句，3-5 人）
├── 可行动洞察（按置信度分层）
└── 方法论反思 / 下次建议（可选）
```

**合成原则**：
- 不做简单拼凑，按逻辑重新组织
- 去重相同内容，保留最完整的版本
- 按重要性排序（洞察 > 数据 > 访谈 > 方法论）

### Step 3: 内容美化

应用 taste-skill 规则：
- **禁止 em-dash (——)**，改用逗号或句号
- **数据高亮**：分数、价格、百分比用加粗或标签
- **洞察编号**：按重要性排序，标注置信度
- **分层展示**：立即行动 vs 验证中 vs 待挖掘

### Step 4: 生成 HTML

输出 taste-skill 风格 HTML：
- 纯 CSS（无外部依赖）
- 蓝紫色主色调（#6366f1）
- KPI 卡片 + 洞察卡片 + 数据表格
- 响应式布局

### Step 5: 保存输出

- MD 报告：`[项目名]报告.md`
- HTML 报告：`[项目名]报告.html`
- 保存到原始文件所在目录

## 设计配置

taste-skill 三档配置（用于 CSS 生成）：

| 配置项 | 值 | 说明 |
|--------|---|------|
| VARIANCE | 5-6 | 报告需稳重，可用适度不对称 |
| MOTION | 3-4 | 轻度交互，悬停效果即可 |
| DENSITY | 5-6 | 数据密集型，保持信息密度 |

## HTML 组件

生成的 HTML 包含：

| 组件 | 用途 | 样式 |
|------|------|------|
| KPI 卡片 | 顶部关键指标（评分、人数、金额） | 4 列 grid，高亮第一个 |
| 洞察卡片 | 关键发现 | 2 列 grid，左边框颜色 |
| 数据表格 | 用户评分、价格区间 | 斑马纹，hover 高亮 |
| 用户金句 | 精选访谈片段 | 头像 + 评分徽章 |
| 行动清单 | 可执行洞察 | 颜色标签（立即/验证/深挖） |

## 触发方式

### 完整触发（推荐）

```
将 xxx 目录的 MD 文件合并为报告 HTML
```

### 带选项

```
将 xxx 目录的 MD 文件合并为报告，要求：
- 突出 [某个主题/指标]
- 包含 [特定用户] 的访谈
- 输出文件命名为 [自定义名称]
```

### 关键词

合并报告 | 生成报告 | 报告 HTML | 访谈合并 | markdown 转报告 | taste-skill 报告

## 示例输入输出

**输入**：目录 `Project/Report/大宝人群访谈分析/` 包含：
- `RoleSim 大宝访谈实验.md`（原始访谈）
- `大宝精华访谈大纲.md`（访谈提纲）
- `RoleSim 大宝访谈实验 1st.md`（方法论分析）

**输出**：
- `大宝精华概念盲测报告.md`
- `大宝精华概念盲测报告.html`

## 注意事项

1. **图片处理**：`![[image.png|409]]` → 转换为标准 Markdown 图片语法
2. **Frontmatter**：优先从 YAML 提取 title、author、description
3. **价格识别**：自动识别 200-600 元等价格区间并高亮
4. **分数识别**：自动识别 3.4 分等评分并分类（高/中/低）
5. **去重**：相同内容出现在多个文件时，保留最完整的版本

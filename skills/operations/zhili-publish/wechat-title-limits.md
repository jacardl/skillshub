# 微信草稿标题字数限制（实测 2026-05）

## 核心发现

微信 `draft/add` API 的 `title` 字段限制是**字符数**而非字节数，但实测阈值远低于理论估算。

| 标题类型 | 理论估算 | 实测通过 | 实测失败 |
|----------|----------|----------|----------|
| 纯中文 | ≤25字节（≈12字符） | ≤10字符 | 12字符（`免费域名这件事，为什么突然变天了`） |
| 中文+英文混合 | ≤28字节（≈16字符） | ≤16字符 | 18字符（`AI写作去机器味GitHub项目7天`） |

## 根因分析

微信对 title 字段有**额外的存储层处理**，导致实际编码消耗高于理论值。「汉字×2字节」的乐观估算不可靠。

## 安全阈值（2026-05 实测）

```
纯中文标题：≤10字符
中文+英文混合：≤16字符
```

含英文时，英文单词每个占1-2字节，中文每个占2-3字节，混合后极易超标。

## 禁用标点（额外占字节）

- 全角冒号 `：`：额外3字节
- 全角顿号 `、`：额外3字节
- em dash `——`：额外4字节

## 预检函数

```python
def safe_truncate_title(title, max_chars=10):
    """微信标题安全截断，pure Chinese ≤10 chars, mixed ≤16 chars"""
    if len(title) <= max_chars:
        return title
    # 截断并加省略号
    return title[:max_chars]  # 不加省略号，避免增加字符

# 调用示例
title = "免费域名这件事，为什么突然变天了"
print(len(title))  # 12 → 需截断到10
```

## 错误响应

```
{"errcode":45003,"errmsg":"title size out of limit hint: [...]"}
```

## 关联错误码

| errcode | 字段 | 原因 |
|---------|------|------|
| 45003 | title | 超过字节限制 |
| 45004 | digest | 超过54字节（≈18混合字符） |
| 45110 | author | 含非法字符，固定填`刘生` |
